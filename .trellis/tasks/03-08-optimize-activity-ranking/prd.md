# 优化活跃度排行榜

## Goal
全面优化活跃度排行榜功能，提升性能、增强功能、改善代码质量。

## Requirements (已确认)

### 1. 活跃度维度扩展
- **保留现有**: 查看图片 +1, 发布图片 +10
- **扩展行为**: 添加评论、点赞、收藏等行为分数
- **分数权重**: 不同行为设置不同权重

| 行为 | 分数 | 说明 |
|------|------|------|
| 查看图片 | 1 | 低权重，高频行为 |
| 发布图片 | 10 | 高权重，核心贡献 |
| 评论图片 | 5 | 中等权重 |
| 点赞图片 | 2 | 低权重 |
| 收藏图片 | 3 | 中低权重 |

### 2. 排行榜类型
- **日榜** (保留): 当日活跃度排名
- **周榜** (新增): 本周活跃度排名
- **月榜** (保留): 本月活跃度排名
- **总榜** (新增): 全部时间活跃度累计

### 3. 性能优化
- **本地缓存**: 使用 Caffeine 缓存排行榜数据
- **用户缓存**: 缓存用户基本信息，减少数据库查询

### 4. 其他功能
- **我的排名**: 用户查询自己在各榜单中的排名

## Acceptance Criteria
- [ ] 日榜、周榜、月榜、总榜均可正常查询
- [ ] 新增活跃行为正确计分
- [ ] 排行榜响应时间 < 50ms (P99)
- [ ] 用户排名查询功能可用
- [ ] 单元测试覆盖率 > 80%

## Technical Design

### 数据结构
```
Redis Key 设计:
- 日榜: activity_rank:day:{yyyyMMdd}
- 周榜: activity_rank:week:{yyyyWW}
- 月榜: activity_rank:month:{yyyyMM}
- 总榜: activity_rank:total
- 用户行为去重: activity_action:{userId}:{yyyyMMdd}
```

### 缓存策略
- 排行榜本地缓存: 30秒 TTL
- 用户信息缓存: 5分钟 TTL
- 缓存一致性: 主动失效 + 定时刷新

### 代码结构
- `ActivityScoreType` enum: 定义活跃行为类型和分数
- `RankTimeEnum` (扩展): DAY, WEEK, MONTH, TOTAL
- `RankServiceImpl`: 核心逻辑实现
- 单元测试覆盖关键路径
